# Бизнес-правила и ограничения

## Роли

| Роль | Код | Права |
|------|-----|-------|
| Администратор | `admin` | Всё, включая редактирование любых тестов и подпись протокола |
| Еж | `ezh` | Создание/редактирование своих тестов, подпись протокола |
| Кот | `kot` | Обучение, экзамен, профиль, черновик PDF; **не** создаёт тесты |

Назначение ролей: только **admin** через API `PUT /api/admin/users/{id}/role` (тело `{"role":"admin"|"ezh"|"kot"}`). Свою роль admin изменить не может. Регистрация всегда создаёт `kot`.

Группа по электробезопасности (ЭБ) назначается staff (`admin`/`ezh`) через `PUT /api/staff/kot-users/{id}/safety-group`. Экзамен и список доступных тестов фильтруются по группе пользователя.

## Экзамен

- Экзамен = **один случайный билет** из пула опубликованных тестов группы ЭБ пользователя.
- Вопросы в билете: **10 случайных** из всех билетов тестов этой группы (состав фиксируется в `attempt.exam_ticket_order`).
- Лимит на **один билет**: 20 минут (`EXAM_TICKET_TIME_LIMIT_SECONDS`).
- Сдача: **≥ 75%** правильных (`MIN_PASS_PERCENT`) — оценка **удовлетворительно** и выше.
- Экзамен сдаётся в присутствии контролирующего (`admin`/`ezh`).
- Подпись протокола — только после успешной сдачи и заполненного профиля Кота.

## Тренировка

- Билеты проходятся по одному; ответы можно отправлять частично.
- Досрочное завершение: в итоговую оценку входят **только билеты, по которым были отправлены ответы** (не начатые билеты не учитываются).

## Тесты

- До **500** билетов на тест, **10** вопросов на билет.
- Тест «готов», когда все билеты полностью заполнены.
- Кот не может создавать тесты (`AccessPolicy`).
- У каждого теста задаётся группа ЭБ (`safety_group`); Кот видит только тесты своей группы.

## Протоколы

| Тип | Путь | Кто может |
|-----|------|-----------|
| Черновик профиля | `/api/profile/protocol.pdf` | только Кот (свой профиль) |
| Черновик пользователя | `/api/admin/users/{id}/protocol-draft.pdf` | только **admin** |
| Черновик по попытке | `…/protocol-draft.pdf` | **admin** и **ezh** |
| Форма по попытке | `…/protocol-form.pdf` | **admin** и **ezh**, после успешной сдачи |
| Метаданные подписанного | `…/protocol` | экзаменуемый, подписант, **admin**/**ezh** |
| PDF подписанного | `…/protocol.pdf` | экзаменуемый, подписант, **admin**/**ezh** |

Подписанный протокол недоступен посторонним пользователям (защита от IDOR): проверяется `examinee_id`, `signer_id` или роль с правом редактирования тестов.

## Экспорт

- PDF протокола и CSV попыток — **асинхронно** (202 + poll).
- Задачи в Redis (prod) или in-memory (dev без Redis).
- Доступ к задаче только владельцу.

## Безопасность

- Лимит неудачных логинов: `LOGIN_RATE_LIMIT_*`.
- Сессия в HttpOnly cookie.
- Rate limit API через Redis (опционально).
- CSRF на мутирующих запросах (`X-CSRF-Token`).
- Подписанные протоколы: авторизация по роли участника экзамена, не только по знанию `attempt_id`.

## Оценки (отображение)

| % | Оценка |
|---|--------|
| &lt; 75 | неудовлетворительно |
| 75–84 | удовлетворительно |
| 85–94 | хорошо |
| ≥ 95 | отлично |

Порог **сдачи** экзамена совпадает с нижней границей «удовлетворительно» (**75%**).
